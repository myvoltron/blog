# Overview
대학교 수업에서 redhat 6.2를 사용하여 실습을 진행하는 경우가 있습니다. 그러나 제 노트북의 경우엔 최신 맥북이라 x86 아키텍처의 CPU가 아니고 그에 따라 VMware와 redhat 6.2를 로컬에 설치할 수 없는 상황입니다. 
그래도 집에 윈도우 데스크탑이 있으니 여기서 redhat 6.2를 설치해서 맥북에서 ssh 접속을 하면 어떨까? 했는데 기본적으로 인터넷에서 설치할 수 있는 redhat 6.2에는 ssh도 안 깔려있고 기본적인 패키지 매니저도 없습니다. 그래서 ssh 접속하기 위한 과정이 까다롭고 관련된 자료가 없었기에, 제가 직접 찾아서 시행착오를 거친 것을 기록하겠습니다. 

우선, 제 환경은 다음과 같습니다.
- 윈도우11 데스크탑
	- VMware
		- Redhat 6.2 - 윈도우 데스크탑에서 VMware를 통해 redhat 6.2를 호스팅합니다.
- Macbook pro m4 - 해당 기기에서 윈도우11 데스크탑의 redhat 6.2로 접속해야합니다. 
	- termius - ssh client

## 4.1 redhat 6.2 재실행
redhat 6.2를 재실행하고 싶은 경우가 있을텐데, 기본적으로 redhat 6.2 접속 시 `linux-up`을 입력해야 redhat 6.2가 켜집니다. 그래서 외부 원격접속한 상태에서 재실행했을 때 `linux-up`을 입력할 수 있는 방법이 없기 때문에, 자동으로 `linux-up`을 통해 redhat 6.2가 켜지도록 설정해야합니다.
